# 苏移舆情预充值风险脚本使用方式

## 1. 功能说明

默认功能：

- 调用苏移舆情专题接口 `THEME_API_URL`
- 专题名称默认使用 `风险研判`
- 拉取负面舆情数据
- 使用本地规则完成地域过滤、场景过滤、风险分类、去重和摘要生成
- 可选调用大模型进行辅助判断
- 输出 Excel 风险清单
- 运行配置从 `.env` 读取；如果 `.env` 不存在，脚本会自动创建模板

说明：下面命令里的脚本路径、输出路径、模型地址、模型名称和模型 key 都是示例。实际运行时请改成你自己机器上的文件地址和模型配置。

## 2. 推荐运行方式

先安装依赖。推荐使用虚拟环境，避免影响系统 Python。

创建虚拟环境：

```bat
你的Python路径 -m venv 你的虚拟环境目录
```

启用虚拟环境：

```bat
你的虚拟环境目录\Scripts\activate
```

使用国内镜像源安装依赖：

```bat
pip install -r 你的requirement.txt路径 -i https://pypi.tuna.tsinghua.edu.cn/simple
```

示例：

```bat
D:\Anaconda\python.exe -m venv D:\你的目录\.venv
D:\你的目录\.venv\Scripts\activate
pip install -r D:\你的目录\requirement.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

也可以使用其他国内镜像源：

```text
清华源: https://pypi.tuna.tsinghua.edu.cn/simple
阿里源: https://mirrors.aliyun.com/pypi/simple/
腾讯源: https://mirrors.cloud.tencent.com/pypi/simple
华为源: https://repo.huaweicloud.com/repository/pypi/simple
```

只使用本地规则，适合快速验证或不准备调用大模型的情况：

```bat
你的Python路径 你的脚本路径 --analysis-mode cpu --output 你的输出Excel路径
```

示例：

```bat
D:\Anaconda\python.exe D:\你的目录\suyiyu_prepaid_risk.py --analysis-mode cpu --output D:\你的目录\预充值卡商户跑路风险预警输出.xlsx
```

CPU 规则优先，大模型辅助复核和智能去重，适合正式运行：

```bat
你的Python路径 你的脚本路径 ^
  --analysis-mode gpu ^
  --model-url 你的大模型接口地址 ^
  --model-key 你的大模型key ^
  --model-name 你的模型名称 ^
  --output 你的输出Excel路径
```

如果要读取本地文件，不调用接口：

```bat
你的Python路径 你的脚本路径 --input 你的输入Excel路径 --analysis-mode cpu --output 你的输出Excel路径
```

## 3. 默认参数

```text
专题名称: 风险研判
情感类型: negative
时间范围: 近 7 天
来源类型: all
每页数量: 20
最大页数: 200
研判模式: cpu
启用 LLM: 否；默认 cpu 模式不调用大模型
LLM 并发数: 4
LLM 调用超时: 120 秒
LLM 正文最大字符数: 5000
存疑线索: 默认写入最终 Excel，提示人工复核
目标城市: 无锡市
目标省份: 江苏省
辖区关键词: 梁溪区,锡山区,惠山区,滨湖区,新吴区,江阴市,宜兴市,经开区,无锡
模糊去重阈值: 0.90
```

`--include-suspected` 默认启用。也就是说，五级不实传言/存疑线索会默认进入最终 Excel，处置标签为“标记存疑，人工复核”。如果不想输出存疑线索，可以加 `--exclude-suspected`。

`--app-key` 和 `--secure-key` 当前脚本里有默认值，也可以通过命令行参数或 `SUYIYU_APP_KEY`、`SUYIYU_SECURE_KEY` 环境变量覆盖。为了安全和便于迁移，正式部署时更推荐用参数或环境变量传入。

注意：脚本不再内置真实密钥。舆情 API 和大模型 API 推荐写入 `.env`；也可以通过命令行参数或环境变量临时覆盖。

## 4. 大模型配置方式

方式一：运行时传参。

```bat
你的Python路径 你的脚本路径 ^
  --analysis-mode gpu ^
  --model-url 你的大模型接口地址 ^
  --model-key 你的大模型key ^
  --model-name 你的模型名称 ^
  --output 你的输出Excel路径
```

方式二：使用环境变量。

```bat
set LLM_API_URL=你的大模型接口地址
set LLM_API_KEY=你的大模型key
set LLM_MODEL=你的模型名称
你的Python路径 你的脚本路径 --analysis-mode gpu --output 你的输出Excel路径
```

方式三：修改脚本内的默认配置。

```text
DEFAULT_LLM_API_URL
DEFAULT_LLM_API_KEY
DEFAULT_LLM_MODEL
```

方式四：使用 `.env` 文件。脚本启动时会自动读取项目根目录的 `.env`，不存在时会自动创建模板。

```text
SUYIYU_APP_KEY=你的舆情app key
SUYIYU_SECURE_KEY=你的舆情secure key
LLM_API_URL=你的大模型接口地址
LLM_API_KEY=你的大模型key
LLM_MODEL=你的模型名称
```

大模型接口必须使用 OpenAI-compatible 的 Chat Completions 协议。这里的 `LLM_API_URL` 要填完整接口地址，不是只填模型网关域名；常见格式如下：

```text
https://你的模型网关/v1/chat/completions
```

有些网关可能使用 `https://你的模型网关/chat/completions`。脚本不会自动补路径，会直接请求你配置的 `LLM_API_URL`；只要该地址支持 `model`、`messages` 入参，并返回 `choices[0].message.content` 即可。

可以先运行连通性测试脚本，测试通过后按提示写入 `.env`：

```bat
python scripts\test_yuqing_api.py
python scripts\test_llm_api.py
```

`.env` 已加入 `.gitignore`，不要提交真实密钥。

如果只想直接导出舆情 API 返回的完整原始结果，不做风险过滤、地域过滤、去重或大模型复核，可以运行：

```bat
python scripts\export_yuqing_api.py ^
  --date-type 30 ^
  --output outputs\舆情API原始结果.xlsx
```

该脚本会使用 `.env` 中的 `SUYIYU_APP_KEY` 和 `SUYIYU_SECURE_KEY`，按页拉取 API 数据，并将返回记录里的全部字段写入 Excel。它适合做原始数据留档、接口排查或和风险筛选结果对账。

注意：舆情 API 当前限制单页数量，默认 `--page-count 20`。脚本会自动翻页直到拉完全部数据，不需要手动逐页运行。如果把 `--page-count` 设置得过大，接口会返回“每页数量超过限制”。

如果希望通过浏览器页面配置参数、运行任务和查看实时日志，可以启动 Web 控制台：

```bat

```

```python
from yuqing_prepaid_risk.service import (
```

```python
result = run_risk_analysis(
    RiskAnalysisOptions(
        analysis_mode="gpu",
        date_type=30,
        output="outputs/预充值卡商户跑路风险预警输出.xlsx",
    ),
    write_output=True,
    include_items=False,

```

原始舆情导出 API，等价于运行 `scripts\export_yuqing_api.py`：

```python
result = export_raw_yuqing(

```

连通性测试 API，等价于运行 `scripts\test_yuqing_api.py` 和 `scripts\test_llm_api.py`：

```python
yuqing_result = test_yuqing_api(YuqingApiTestOptions())
llm_result = test_llm_api(LlmApiTestOptions())
```

服务层 API 默认读取 `.env`。如果外部系统自己管理密钥，可以在 Options 里直接传入 `app_key`、`secure_key`、`model_url`、`model_key`、`model_name` 等参数。`run_risk_analysis(..., include_items=True)` 会返回每条处理后的明细；如果只需要生成 Excel，建议设为 `False`，减少内存占用。

如果模型接口不可用，或只想使用本地规则，可以使用：

```bat
你的Python路径 你的脚本路径 --analysis-mode cpu --disable-llm --output 你的输出Excel路径
```

`--disable-llm` 的作用是强制关闭大模型辅助判断。即使传了 `--model-url`、`--model-key`、`--model-name`，或者使用了 `--analysis-mode gpu`，只要同时加了 `--disable-llm`，脚本就不会调用大模型，实际效果会退回本地规则判断。

## 5. 两种模式和工作流程

### cpu 模式

调用方式：

```bat
你的Python路径 你的脚本路径 --analysis-mode cpu --output 你的输出Excel路径
```

处理逻辑：

1. 拉取接口数据，或读取 `--input` 指定的本地文件。
2. 使用本地规则判断地域、预充值关键词、跑路/闭店关键词、行业场景、普通消费纠纷、过期旧闻、疑似传言等。
3. 对本地规则认为需要保留的记录做 URL 精准去重、全文精准去重、同门店同事件归并、相似文案模糊去重。
4. 不调用大模型。
5. 输出最终保留记录；如果加 `--include-filtered`，也会输出被过滤和去重的记录，便于审计。

适用场景：

- 只想快速跑结果。
- 没有可用大模型接口。
- 希望结果完全由固定规则生成。

### gpu 模式

调用方式：

```bat
你的Python路径 你的脚本路径 ^
  --analysis-mode gpu ^
  --model-url 你的大模型接口地址 ^
  --model-key 你的大模型key ^
  --model-name 你的模型名称 ^
  --output 你的输出Excel路径
```

处理逻辑：

1. 先执行 CPU 模式的本地规则初筛。
2. 明显外地、明显重复、明显无关的记录直接过滤，不再调用大模型；外地城市关键词覆盖全国地级市和直辖市，并排除目标城市无锡。
3. CPU 初判保留的高风险记录，会交给大模型复核。
4. CPU 初判为存疑传言，或虽然初步过滤但同时具备预充值和跑路风险特征的记录，也会交给大模型复核。
5. 大模型返回后，可以把记录改为保留，也可以改为过滤。
6. 模型复核完成后，再对最终保留记录执行一次 URL、全文、同门店同事件和相似文案去重，避免模型改回保留的记录绕过去重。
7. 对规则仍未归并的候选组，再用大模型辅助判断是否属于同一事件、同一门店多用户投诉、相似话术投诉或同一行业综述转述。
8. 最终输出以规则和大模型综合后的 `keep` 结果为准。

适用场景：

- 推荐正式运行使用。
- 希望控制模型调用量。
- 希望明显无关和重复内容先由规则剔除，有风险的内容再让大模型辅助判断。
- 希望对“同一事件不同标题转述”“同一门店多用户投诉”“相似话术投诉”做更强语义归并。

## 7. 输出字段说明

输出 Excel 包含原始舆情字段和研判字段。

主要研判字段：

```text
risk_level      风险等级
risk_label      处置标签
risk_summary    模型或规则生成的摘要
dedup_key       去重键
dedup_reason    去重或保留原因
model_used      是否调用大模型
model_reason    大模型判定依据；模型失败时会写失败原因并回退规则
```

重点字段解释：

- `dedup_key`：脚本内部用于去重/归并的键。常见形式包括门店/公司+事件+金额、标题+地点+事件、URL、全文哈希、模型归并保留记录 id 等。它主要用于判断多条舆情是否指向同一事件。
- `dedup_reason`：这条记录最终为什么保留或剔除。常见值包括 `保留`、`地域过滤去重`、`全文精准去重`、`URL精准去重`、`同门店同事件归并`、`相似文案模糊去重:#序号`、`最终同门店同事件归并`、`最终相似文案模糊去重:#序号`、`大模型同事件归并:#保留记录id`、`不符合预充值跑路场景`、`未体现预充值办卡风险`、`普通消费纠纷`、`直接过滤剔除` 等。
- `model_used`：是否调用过大模型复核，值为 `是` 或 `否`。为 `否` 时表示只用了本地规则结果，或该记录属于明显去重/过滤记录，不再送模型。
- `model_reason`：大模型给出的判定理由。如果模型调用失败，会写入失败原因，并使用本地规则结果。

风险等级包括：

```text
一级真实高风险负面
二级普通消费
三级无效水帖 / 吐槽
四级过期旧闻
五级不实传言
```

## 8. 过滤和去重逻辑说明

地域过滤：

- 判断标题和正文是否出现目标城市、目标城市简称、辖区关键词。
- 判断 `iplocation` 是否属于目标省份或目标城市。
- 如果标题或正文出现脚本内置的外市关键词，且没有出现目标城市或辖区关键词，会判为 `地域过滤去重`。
- 外市关键词已覆盖全国地级市和直辖市，江苏省内排除目标城市 `无锡`。例如标题明确出现 `兰州仁和医院`，且没有无锡辖区主事件，会直接按地域过滤剔除。

风险场景过滤：

- `不符合预充值跑路场景`：没有同时形成“预充值/办卡/会员卡/储值卡/课包/余额”等预付消费要素，以及“关门/闭店/跑路/失联/拒不退款/退款难”等风险要素。
- `未体现预充值办卡风险`：命中了跑路、闭店、退款难等风险词，但没有体现预付、充值、办卡、会员卡、储值卡等预付消费要素。
- `普通消费纠纷`：只体现服务差、态度差、体验差、预约难、乱收费、价格贵等普通投诉，没有闭店跑路风险。
- `过期旧闻`：出现已解决、已处置、已完结、已经退款、达成和解、历史旧闻等时效过滤词。
- `五级不实传言`：出现听说、疑似、好像、传言、网传、猜测等存疑表达，同时具备商户和跑路风险特征。

去重逻辑：

- `URL精准去重`：URL 完全重复。
- `全文精准去重`：标题和正文规范化后完全重复。
- `同门店同事件归并`：抽取门店/公司/平台主体、事件词和金额，生成同一个事件键时归并。例如 `无锡五八悦家信息技术有限公司 + 失联 + 199元` 会归并为同一事件。
- `相似文案模糊去重`：标题和正文规范化后计算文本相似度，默认相似度达到 `0.90` 即归并。
- `最终...去重`：gpu 模式下，大模型风险复核完成后，会对所有最终保留记录再跑一次规则去重，防止模型把 CPU 初筛过滤的重复记录改回保留。
- `大模型同事件归并`：gpu 模式下，规则去重后仍保留的候选组会交给大模型辅助判断。候选组会按同标题、同主体、同门店/公司、同主题事件归组，例如烘焙行业闭店潮、顶东面包花园、58旺铺会员服务退款投诉等。

大模型辅助去重会要求模型先提取：

- 主体/品牌/门店/公司/平台
- 地点
- 行业
- 预付费问题
- 闭店/失联/拒不退款事实
- 时间线

然后判断是否属于同一事件不同转述、同一门店多用户投诉、相似话术投诉，或同一行业综述反复转述同一批案例和同一组数字。

模糊去重算法简要说明：

1. 把标题和正文拼接在一起。
2. 删除 URL、空白字符和标点符号，只保留中文、英文、数字等主要文本。
3. 取规范化后的前 1200 个字符作为比较文本。
4. 先用前 300 个字符做快速相似度预筛，明显不相似的直接跳过。
5. 对通过预筛的文本，使用 Python 标准库 `difflib.SequenceMatcher` 计算整体相似度。
6. 如果整体相似度大于等于 `--fuzzy-threshold`，默认 `0.90`，就判为 `相似文案模糊去重`。

## 9. 运行日志和统计指标说明

脚本运行时会打印接口拉取、大模型任务和最终统计。例如：

```text
[2026-05-21 17:32:43] 舆情接口获取完成: theme=风险研判, rows=195
[2026-05-21 17:32:43] 大模型任务准备完成: mode=gpu, tasks=68, workers=4
[2026-05-21 17:32:58] 大模型辅助去重任务准备完成: groups=6
```

含义：

- `舆情接口获取完成: theme=风险研判, rows=195`：已从苏移舆情 `风险研判` 专题接口拉取完成，共获取 195 条原始舆情。
- `大模型任务准备完成: mode=gpu, tasks=68, workers=4`：当前有 68 条记录准备调用大模型辅助风险判断；`workers=4` 表示最多 4 个并发线程调用模型。
- `大模型辅助去重任务准备完成: groups=6`：规则去重后仍有 6 个候选组需要大模型判断是否归并。

最终统计示例：

```text
输入总量: 195
输出总量: 41
最终保留条数: 41
CPU去重条数: 1
CPU风险过滤条数: 153
不符合预充值跑路场景: 51
保留高风险: 41
全文精准去重: 1
地域过滤去重: 46
大模型调用量: 68
普通消费纠纷: 2
未体现预充值办卡风险: 27
直接过滤剔除: 27
输出文件: 你的输出Excel路径
```

各项含义：

- `输入总量`：接口或本地文件读到的原始舆情总数。
- `输出总量`：最终写入正式结果清单的记录数，即脚本判断需要保留的高风险记录数。
- `最终保留条数`：最终保留的记录数，通常与 `输出总量` 一致。
- `CPU去重条数`：被去重逻辑剔除的记录数，包括 URL 精准去重、全文精准去重、同门店同事件归并、相似文案模糊去重、最终去重和大模型同事件归并等。
- `CPU风险过滤条数`：被过滤掉、且不属于去重的记录数，例如地域不符、普通消费纠纷、未体现预充值办卡风险、场景不符合等。
- `保留高风险`：最终保留下来的高风险预警记录数，通常与 `输出总量` 一致。
- `不符合预充值跑路场景`：没有同时满足预充值/办卡/会员卡等关键词和跑路/闭店/失联/拒不退款等风险场景。
- `地域过滤去重`：被判断为外市、非目标辖区或与目标地区无关的舆情。
- `普通消费纠纷`：只有服务差、态度差、体验差、价格争议等普通消费问题，没有闭店跑路风险。
- `未体现预充值办卡风险`：可能提到跑路、闭店、退款难等，但没有体现预付、充值、办卡、会员卡、储值卡等预付消费风险。
- `直接过滤剔除`：大模型或规则给出的通用过滤结论，表示不建议进入预警清单。
- `全文精准去重`：标题和正文规范化后完全重复，因此剔除。
- `大模型调用量`：实际调用大模型复核的记录数。它不是输出数或过滤数的加总，而是表示有多少条记录经过模型判断。
- `输出文件`：最终生成的 Excel 文件路径。

数量关系一般可以这样理解：

```text
输入总量 = 输出总量 + 被过滤/去重数量
被过滤/去重数量 = CPU风险过滤条数 + CPU去重条数
```

以上面示例为例：

```text
195 = 41 + 154
154 = 153 + 1
```
